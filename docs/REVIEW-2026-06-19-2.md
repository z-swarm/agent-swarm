## 3. 严重风险(P0 — 必修)

### 3.1 🔴 H1 — `channels/lark.py` 签名验证完全失效

**问题位置**: `src/agent_swarm/channels/lark.py:55-63`(`_hmac_sha256_hex`)

```python
def _hmac_sha256_hex(key: str, payload: str) -> str:
    """
    飞书事件签名核心:HMAC-SHA256 hex
    @note 飞书官方规范: digest = sha256(timestamp + nonce + token + encrypt_key + body)
          这里简化为 sha256(timestamp + nonce + body),encrypt_key 为可选字段
    """
    h = hashlib.sha256()                        # ← 不是 HMAC!
    h.update(payload.encode("utf-8"))           # ← key 完全没用上
    return h.hexdigest()
```

**双重 bug**:
1. **函数名 "HMAC" 误导** —— 实现是 `hashlib.sha256()`,**不是 HMAC**
2. **`key` 参数被丢弃** —— key 接收了但完全没参与计算

更糟糕的是 `verify_lark_signature` 把 `verification_token` 既未作为 HMAC key,也未正确 concat 进 payload:

```python
# lark.py:91-95
if encrypt_key:
    payload = f"{timestamp}{nonce}{encrypt_key}{body}"  # token 没进 payload!
else:
    payload = f"{timestamp}{nonce}{body}"               # token 也没进 payload!
return _hmac_sha256_hex(verification_token, payload)    # key 又被丢弃
```

**实际效果**: `sha256(timestamp + nonce + body)` —— **完全没用 token**。

### 探针验证(已实跑)

```
=== Test 1: _hmac_sha256_hex 用不同 key 产生不同签名? ===
  sig_a (correct-token): 51fa92cb46edfbe5...
  sig_b (wrong-token):   51fa92cb46edfbe5...     ← 完全相同!
  sig_c (empty):         51fa92cb46edfbe5...     ← 完全相同!

=== Test 2: 攻击者(不知道 token)能否伪造签名? ===
  攻击者算出的 sig:  51fa92cb46edfbe5...
  服务端算出的 sig:  51fa92cb46edfbe5...
  两者相同? True  ← 攻击成功!

=== Test 3: 真实场景 ===
  服务端校验的签名 = sha256(timestamp + nonce + body)
  攻击: 不需要知道 token,直接 sha256(timestamp + nonce + body) 就能算
```

### 安全影响

- **飞书通道整个信任链失效**(签名 + 白名单 + 限流全部绕过)
- **W11 ChannelApprover 审批回调也受影响**(走同一签名路径)
- 任何能访问 webhook URL 的人:
  - 伪造任意 `open_id` → 注入消息到 swarm
  - 触发任意审批卡片回调(approve/deny)
  - 注入恶意 task → 触发 agent 执行
- 已部署的飞书实例:**立即审计 webhook 访问日志**,看是否有未授权请求成功

### 修复方案

```python
def _hmac_sha256_hex(key: str, payload: str) -> str:
    import hmac, hashlib
    return hmac.new(
        key.encode("utf-8"),
        payload.encode("utf-8"),
        hashlib.sha256,
    ).hexdigest()

# verify_lark_signature 改为:
def verify_lark_signature(timestamp, nonce, body, verification_token, encrypt_key=None) -> str:
    # 飞书官方 v2 规范:
    # payload = timestamp + nonce + encrypt_key(可选) + body
    # sig = HMAC-SHA256(key=verification_token, message=payload)
    payload = f"{timestamp}{nonce}{encrypt_key or ''}{body}"
    return _hmac_sha256_hex(verification_token, payload)  # key 真正作为 HMAC key
```

**测试侧必须新增**:
- `test_verify_lark_signature_changes_with_token`:断言不同 token 产生不同签名
- `test_verify_lark_signature_rejects_forged_timestamp`:攻击者伪造 timestamp + nonce + body 应被拒
- `test_hmac_uses_key_not_concat`:断言实现使用 hmac.new() 而非 hashlib.sha256()

### 根因复盘

W10 落地时引入了**测试假象**:

| 测试 | 实际验证 | 应该验证 |
|---|---|---|
| `test_verify_lark_signature_basic` | 相同输入 → 相同输出(平凡) | **不同 token → 不同签名**(核心安全属性) |
| `test_verify_lark_signature_with_encrypt_key` | 有/无 encrypt_key 区分 | token 作为 key 必须产生不同签名 |

**铁律**:任何"加密/签名/认证"代码,**第一件事是写"不同输入产生不同输出"的回归测试**。"相同输入产生相同输出"只是哈希函数的基础属性,不能保证安全。

## 4. 中风险(P1 — 应修)

### 4.1 🟡 M1 — `tools/agent_review.py` PATH_TRAVERSAL 规则既漏一半又误报一半

```python
# tools/agent_review.py:140-145
{
    "category": "PATH_TRAVERSAL",
    "severity": "HIGH",
    "pattern": re.compile(r"open\(\s*[^)]*(\+\s*[a-zA-Z_])"),
    "description": "open() 参数含字符串拼接 — 可能存在 path traversal",
},
```

**问题**:
- **漏报**:`open(f"path/{name}")` f-string 写法不被命中(实际项目更常见)
- **误报**:`open(p + ext)` 这种无害的扩展名拼接也会被命中

**建议**:改为真正检测不可信输入拼接:

```python
# 检测不可信来源的路径拼接
"pattern": re.compile(
    r"open\(\s*[^)]*"
    r"(?:\+\s*(?:user_Input|request\.|input\(|argv\[|args\[|\.params\[))"
),
```

### 4.2 🟡 M2 — `tools/agent_review.py` WEAK_HASH 误报 md5 非安全用途

```python
# tools/agent_review.py:167-170
{
    "category": "WEAK_HASH",
    "severity": "MEDIUM",
    "pattern": re.compile(r"hashlib\.(md5|sha1)\b"),
    ...
}
```

**问题**: `hashlib.md5` 在 Python 里常用作:
- 文件指纹 / etag
- 缓存 key / idempotency key
- 内容去重

直接报 MEDIUM 会让所有静态扫描 PR 都挂一堆误报,**消磨审查者注意力**,反而漏掉真正的问题。

**建议**:加场景识别 — 在 dedup / cache / etag / idempotency 上下文里降级为 LOW 或跳过。

### 4.3 🟡 M3 — `tools/agent_review.py` `lines_changed` 统计不准

```python
# tools/agent_review.py:111-114
for line in diff.splitlines():
    if line.startswith("+") and not line.startswith("+++"):
        lines += 1
```

只数加行,不数减行,跟 git diff --numstat 行为不一致。

**建议**:直接解析 `--numstat`:

```python
result = subprocess.run(
    ["git", "diff", pr_ref, "--numstat"], ...
)
# 输出格式: "5\t3\tsrc/foo.py" → 5 加 / 3 减
```

## 6. 测试盲点

### 6.1 `test_verify_lark_signature_basic` 等测试只测"相同输入→相同输出"

```python
def test_verify_lark_signature_basic() -> None:
    sig1 = verify_lark_signature("1700000000", "nonce1", '{"x":1}', "token-abc")
    sig2 = verify_lark_signature("1700000000", "nonce1", '{"x":1}', "token-abc")
```
